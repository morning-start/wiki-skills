# 需求追踪矩阵（RTM）模板

## 概述

需求追踪矩阵（Requirements Traceability Matrix, RTM）用于确保需求、设计、实现、测试之间的完整追溯，保障文档一致性。

### 用途

- **正向追踪**：需求 → 设计 → 实现 → 测试
- **反向追踪**：测试 → 实现 → 设计 → 需求
- **覆盖度分析**：识别未实现的需求、未测试的功能
- **变更影响分析**：评估需求变更的影响范围

### 使用说明

1. 为每个需求分配唯一 ID（如 REQ-001）
2. 填写需求描述和优先级
3. 关联设计文档章节
4. 关联实现模块/代码
5. 关联测试用例
6. 跟踪状态变化

### 字段说明

- **需求 ID**：唯一标识符（格式：REQ-XXX）
- **需求描述**：简要描述需求内容
- **优先级**：P0（最高）/ P1（高）/ P2（中）/ P3（低）
- **设计章节**：设计文档中的对应章节编号
- **实现模块**：代码中的模块/类/方法
- **测试用例**：相关测试用例 ID（TC-XXX）
- **状态**：待开始 / 进行中 / 已完成 / 已验证 / 已取消
- **备注**：其他说明信息

---

## 状态定义

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| 待开始 | 需求尚未开始实施 | 需求评审通过 | 设计开始 |
| 进行中 | 需求正在实施 | 设计开始 | 代码完成 |
| 已完成 | 需求已实现 | 代码完成并通过单元测试 | 测试开始 |
| 已验证 | 需求已通过测试 | 测试用例执行通过 | 发布 |
| 已取消 | 需求被取消 | 需求评审不通过/变更 | - |

## 最佳实践

### 1. 及时更新

- 需求变更时立即更新 RTM
- 每个迭代结束时审查 RTM
- 版本发布前验证 RTM 完整性

### 2. 工具支持

- 使用 Excel/Google Sheets 管理 RTM
- 集成 Jira/禅道等需求管理工具
- 自动化生成覆盖度报告

### 3. 评审机制

- 需求评审时检查 RTM 完整性
- 设计评审时验证设计覆盖率
- 测试评审时确认测试覆盖率

### 4. 变更控制

- 所有需求变更必须更新 RTM
- 评估变更影响范围
- 跟踪变更实施状态
