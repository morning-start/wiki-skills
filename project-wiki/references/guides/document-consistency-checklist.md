# 文档一致性检查清单

## 概述

本文档提供系统性的文档一致性检查方法，涵盖基础文档清单和 7 大类一致性检查内容，帮助团队保障文档质量。

### 核心目标

- **同一事实，在所有相关文档中表述相同**
- **逻辑自洽，需求 - 设计 - 实现 - 测试对齐**
- **版本同步，所有文档版本号、状态一致**

### 适用场景

- 文档生成后的质量检查
- 文档更新后的一致性验证
- 版本发布前的文档审核
- 项目里程碑评审

## 二、术语与命名一致性检查

### 2.1 技术术语一致性

**检查目标**：确保技术术语在全文中使用一致

**检查项**：
- [ ] 首次出现的术语是否有明确定义
- [ ] 同一术语在不同文档中的定义是否一致
- [ ] 是否存在同义词混用（如"模块"vs"组件"）
- [ ] 全文是否统一使用同一术语
- [ ] 是否存在中英文混用（如"API"vs"接口"）
- [ ] 缩写首次出现是否注明全称

**检查方法**：
```bash
# 搜索术语使用情况
grep -r "术语" --include="*.md" .
```

## 三、内容逻辑一致性检查

### 3.1 需求 - 设计 - 实现对齐

**检查目标**：确保需求、设计、实现三者对齐

**检查项**：
- [ ] 每个需求都有对应的设计文档
- [ ] 每个设计都有对应的实现
- [ ] 需求变更是否同步到设计和实现
- [ ] 功能描述在各文档中是否一致

**检查方法**：
- 使用需求追踪矩阵（RTM）
- 交叉引用检查

---

## 四、版本一致性检查

### 4.1 版本号对齐

**检查目标**：确保版本号在所有文档中一致

**检查项**：
- [ ] README.md 中的版本号
- [ ] CHANGELOG.md 中的最新版本
- [ ] SKILL.md 中的版本号
- [ ] Git Tag 版本号

**检查方法**：
```bash
# 检查各文档版本号
grep "version" README.md
grep "## \[" CHANGELOG.md
grep "version:" SKILL.md
git tag -l
```

## 五、格式与结构检查

### 5.1 文档格式规范

**检查项**：
- [ ] 标题层级清晰（H1 → H2 → H3）
- [ ] 代码块添加语言标签
- [ ] 表格格式正确
- [ ] 链接有效（无 404）
- [ ] 图片有替代文本

**检查工具**：
- markdownlint
- markdown-link-check

## 检查流程

### 快速检查（5 分钟）

1. 检查基础文档是否齐全（P0 文档）
2. 检查版本号一致性
3. 抽查术语使用

### 完整检查（15 分钟）

1. 执行快速检查
2. 检查需求 - 设计 -实现对齐
3. 检查格式与链接
4. 生成检查报告

### 版本发布前检查（30 分钟）

1. 执行完整检查
2. 7 大类检查项逐项检查
3. 生成完整性报告
4. 修复所有问题

---

## 检查报告模板

```markdown
# 文档一致性检查报告

**检查日期**: YYYY-MM-DD
**检查范围**: README.md, ARCHITECTURE.md, CHANGELOG.md
**检查人员**: XXX

## 检查结果

- ✅ 术语与命名：通过
- ✅ 内容逻辑：通过
- ⚠️ 版本一致性：发现 1 个问题
  - CHANGELOG.md 中版本号为 v1.0.0，README.md 中为 v1.0.1
- ✅ 格式与结构：通过

## 需要修复的问题

1. [严重] CHANGELOG.md 版本号不一致
2. [建议] 更新过期的链接

## 总体评价

文档一致性良好，需修复 1 个严重问题后即可发布。
```
