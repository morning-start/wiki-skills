# 文档评审 Checklist

## 使用说明

本 checklist 用于文档评审，确保文档质量。适用于关键文档（架构设计、API 合约、需求文档等）的正式评审。

**评审方式**：
- **异步评审**：PR/MR 中附带文档变更，团队 Review
- **同步评审**：设计评审会议（DR Meeting）

## 使用提示

1. **评审前**：评审人员提前阅读文档，标注问题
2. **评审中**：逐项检查 checklist，记录问题
3. **评审后**：作者修改，评审组长确认
4. **跟踪**：使用版本跟踪表记录修改历史
