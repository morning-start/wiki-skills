## 3. 功能需求

> 按模块或子系统组织功能需求，每个需求应有唯一编号

### 3.1 模块一：用户管理

#### REQ-001：用户注册
- **输入**：邮箱、密码、验证码
- **处理逻辑**：
  1. 验证邮箱格式
  2. 检查邮箱是否已注册
  3. 加密存储密码
  4. 发送激活邮件
- **输出**：注册成功/失败提示
- **异常处理**：
  - 邮箱格式错误 → 提示"邮箱格式不正确"
  - 邮箱已存在 → 提示"该邮箱已注册"

#### REQ-002：用户登录
- **输入**：邮箱/手机号、密码
- **处理逻辑**：
  1. 验证用户存在性
  2. 验证密码正确性
  3. 生成访问令牌
- **输出**：访问令牌、用户信息
- **异常处理**：
  - 用户不存在 → 提示"用户不存在"
  - 密码错误 → 提示"密码错误"
  - 账户锁定 → 提示"账户已锁定，请联系管理员"

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 | 测量方法 |
|------|------|---------|
| 响应时间 | API 平均响应时间 ≤ 500ms，P99 ≤ 2000ms | 压力测试 |
| 吞吐量 | 支持 1000 并发用户 | 负载测试 |
| 可用性 | 99.9%（每月允许停机时间约 43 分钟） | 监控统计 |

### 4.2 安全需求

| 需求项 | 具体要求 |
|--------|---------|
| 身份认证 | 支持多种认证方式（用户名密码、OAuth2.0、SSO） |
| 数据加密 | 传输层使用 HTTPS（TLS 1.3），敏感数据存储使用 AES-256 加密 |
| 防护措施 | 防 XSS、CSRF、SQL 注入，实施速率限制 |
| 审计日志 | 记录所有关键操作（登录、数据修改、权限变更） |

### 4.3 可靠性需求
- 系统崩溃后自动恢复时间 < 5 分钟
- 数据备份：每日全量备份，每小时增量备份
- 故障切换时间 < 10 秒

### 4.4 可维护性需求
- 代码覆盖率 ≥ 80%
- API 接口版本化
- 日志规范：统一日志格式和级别

## 5. 接口需求

### 5.1 用户接口

#### Web 端
- 支持主流浏览器（Chrome、Firefox、Safari）最新 2 个版本
- 屏幕分辨率：1280×720 及以上
- 支持深色/浅色主题

#### 移动端
- iOS 14+ / Android 10+
- 屏幕适配：支持 iPhone SE 至 iPad Pro，Android 各种屏幕尺寸
- 支持触摸手势操作

#### 桌面端
- Windows 10/11
- macOS Monterey+
- 支持快捷键操作

#### 游戏端（如适用）
- 支持手柄/键盘/鼠标输入
- 帧率 ≥ 60fps
- 支持屏幕分辨率切换

### 5.2 系统接口

#### 外部 API 调用
```
接口名称：获取用户信息
请求方式：GET
请求路径：/api/v1/users/{user_id}
请求头：
  - Authorization: Bearer {token}
响应体：
{
  "code": 0,
  "message": "success",
  "data": {
    "user_id": "12345",
    "username": "张三",
    "email": "zhangsan@example.com"
  }
}
错误码：
  - 401：未授权
  - 404：用户不存在
```

### 5.3 数据接口
- 数据库连接池：最大连接数 100，最小连接数 10
- 缓存策略：Redis 缓存热点数据，TTL 1 小时

## 6. 数据需求

### 6.1 数据实体

#### 用户表（users）
| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| user_id | VARCHAR(64) | PK | 用户唯一标识 |
| username | VARCHAR(50) | NOT NULL | 用户名 |
| email | VARCHAR(100) | UNIQUE | 邮箱 |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希值 |
| created_at | TIMESTAMP | NOT NULL | 创建时间 |

### 6.2 数据字典
> 详细定义系统使用的数据项

| 数据项名称 | 类型 | 长度 | 取值范围 | 说明 |
|-----------|------|------|---------|------|
| 用户状态 | INT | 1 | 0-禁用, 1-正常, 2-锁定 | 用户账户状态 |

### 6.3 数据备份与恢复
- 全量备份：每日 02:00 执行

- 恢复演练：每月 1 次

## 7. 质量属性

### 7.1 可扩展性
- 采用微服务架构，支持水平扩展
- 数据库分库分表策略
- CDN 加速静态资源

### 7.2 兼容性
- API 版本化，向后兼容至少 2 个版本
- 多端数据同步策略：
  - 在线优先：最新数据存储于服务端
  - 离线缓存：本地缓存最近操作记录
  - 冲突解决：以服务端数据为准，本地修改标记为待同步

### 7.3 可移植性
- 容器化部署（Docker + Kubernetes）
- 配置与代码分离

## 8. 约束与假设

### 8.1 技术约束
- 后端语言：Java 17 / Python 3.9+
- 前端框架：React 18+ / Vue 3+
- 数据库：MySQL 8.0+
- 消息队列：Kafka / RabbitMQ

### 8.2 业务约束
- 必须符合 GDPR 数据保护法规
- 敏感数据存储在境内服务器
- 用户数据删除请求必须在 30 天内完成

### 8.3 假设
- 用户设备支持现代浏览器
- 网络连接稳定，带宽 ≥ 10Mbps
- 第三方服务可用性 ≥ 99.5%
