# 质量指标定义

本文档定义文档质量评分的维度、权重、计算方法和阈值。

## 评分维度

### 1. 格式规范（10%）

检查项：
- [ ] YAML front matter 格式正确
- [ ] 必填字段存在
- [ ] 标题层级正确（不跳级）
- [ ] 代码块标注语言
- [ ] 表格格式正确
-  列表格式一致

评分规则：
- 全部通过：1.0
- 每错一项扣 0.2
- 最低 0.0

### 2. 一致性（25%）

检查项：
- [ ] 术语使用一致
- [ ] 数据在各文档中一致
- [ ] 版本号在各文档中一致
- [ ] 格式风格一致

评分规则：
- 问题数 0：1.0
- 问题数 1-2：0.8
- 问题数 3-5：0.6
- 问题数 >5：0.4

### 3. 完整性（25%）

检查项：
- [ ] 必需章节完整
- [ ] 示例完整
- [ ] 错误码/状态码完整
- [ ] 多端差异说明（如适用）

评分规则：
- 缺失章节数 / 必需章节总数 = 缺失率
- score = 1.0 - 缺失率

### 4. 追溯完整性（20%）

检查项：
- [ ] depends_on 指向的文档存在
- [ ] feeds_into 指向的文档存在
- [ ] 双向追溯关系对应
- [ ] 追溯链无断点
- [ ] 追溯链无循环

评分规则：
- 追溯关系完整率 = 正确追溯数 / 应追溯数
- score = 追溯关系完整率

### 5. 图示覆盖（20%）

检查项：
- [ ] 架构文档有架构图
- [ ] 复杂流程有时序图
- [ ] 数据模型有 ER 图
- [ ] 状态机有状态图

评分规则：
- 应有图示数 / 实际图示数 = 覆盖率
- score = min(1.0, 覆盖率)

## 综合评分计算

```
overall = format * 0.10 + consistency * 0.25 + completeness * 0.25 + traceability * 0.20 + diagram * 0.20
```

## 状态判定

| overall | status | 处理 |
|---------|--------|------|
| ≥ 0.90 | pass | 通过，无问题 |
| 0.80 - 0.89 | pass | 通过，有改进建议 |
| 0.70 - 0.79 | warning | 标注警告 |
| < 0.70 | fail | 不通过 |

## 自动降级规则

以下情况自动降级为 fail，无论分数如何：

- 存在 error 级别的一致性问题
- 追溯关系有循环
- 必填 YAML 字段缺失
- 必需章节缺失超过 2 个

## 质量报告格式

```yaml
quality_report:
  doc_id: "REQ-001"
  version: "1.0.0"
  overall: 0.87
  status: pass
  breakdown:
    format: 1.00
    consistency: 0.92
    completeness: 0.85
    traceability: 0.88
    diagram: 0.80
  issues:
    - dimension: consistency
      type: terminology
      severity: warning
      detail: "..."
  suggestions:
    - "统一使用'用户'而非'User'"
    - "补充非功能需求章节"
```
